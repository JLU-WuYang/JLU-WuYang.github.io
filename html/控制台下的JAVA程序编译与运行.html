控制台下的JAVA程序编译与运行

类别：JAVA

之前一直在IDE环境下，编写包与类运行正常，但到控制台下出现各种问题，比如未识别的标识符，找不到符号， 类载入错误的情况。
解决方法：
  • 一个文本里只能写一个主类的（即public class XXX），分开写几个类又或则将其他类（除主类的其他类）的public去掉
  • 将几个类都编译，用javac *.java，然后java 主类
原因：
  • 每个编译单元（文件）都只能有一个public类。这表示，每个编译单元都有单一的公共接口，用public类来表现。该接口可以按要求包含众多的支持包访问权限的类。如果在某个编译单元内有一个以上的public类，编译器就会给出出错信息。
关于文件名与类名：
  • Java保存的文件名必须与类名一致
  • 如果文件中只有一个类，文件名必须与类名一致
  • 一个Java文件中只能有一个public类
  • 如果文件中不止一个类，文件名必须与public类名一致
  • 如果文件中不止一个类，而且没有public类，文件名可与任一类名一致
关于编码 GBK 的不可映射字符错误：
  • 首先记事本打开java源文件
  • 然后另存为，选择ANSI编码
  • 覆盖，编译
